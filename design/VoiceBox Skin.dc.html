04 — FOCUS MODEL & HIERARCHY

One focus, one recipe. Exactly one element owns focus at a time — a settings row, a tile, or a strip button. Focus is always the same recipe: 1px accent ring + accent tint fill (2-stop) + the single glow shadow. Nothing else in the UI uses the glow, so the focused element is findable at a glance from across the room. Left/right pane switch is a single keypress; the highlight physically slides (120ms) instead of teleporting, so the eye never loses it.

State is color-coded, not shape-coded. Cyan = active/selected, amber = paused/warning, red = error/clip, gray = off. Hover only lifts surfaces one step and never uses accent, so keyboard focus and mouse hover can coexist without ambiguity. The < > arrows exist only on the focused row — they double as the "this row is adjustable" affordance.

Hierarchy. Labels are Space Grotesk (human words); every value, badge, hotkey, and readout is JetBrains Mono (machine data) — the two-font split is the entire information hierarchy. Values that differ from default render bright; "off" stays faint, so a scan down the list reads like a channel strip: you see what's engaged. Section headers are deliberately quiet hairlines — wayfinding, not content.
